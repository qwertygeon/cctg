# CLI 출력 다국어화(i18n) 설계

> CCTG의 CLI 출력을 **영문/한글로 분리**하고, **설치 시 선택**하며 **설치 후에도 전환**할 수 있게 한 설계 기록.

> **구현 상태: 적용 완료.** `messages/{en,ko}.sh` 카탈로그 + `cc-tg.sh` 의 `t()`/언어 결정, `cctg lang`, `install.sh --lang`, 사용자 설정 `~/.config/cctg/config` 로 본 설계가 구현되었다. 아래는 설계 근거 기록이다.
>
> **언어 중립으로 남긴 항목(follow-up):** 생성되는 `launch.env` 템플릿 주석(파일 내용), `${1:?...}` 필수 인자 누락 에러(셸 builtin), zsh 자동완성 설명(완성 시점 카탈로그 source 복잡도). 카탈로그 누락 시 `t()` 는 `!KEY!` 로 폴백하고 stderr 경고를 낸다.

## 목표

1. 모든 사용자 노출 출력(`status`/`doctor`/`add`/에러 메시지/`usage` 등)을 **언어별 카탈로그**로 분리한다.
2. **설치 시** 언어를 고를 수 있다(미지정 시 로케일 자동 감지, 최종 폴백 영문).
3. **설치 후** 언제든 언어를 바꿀 수 있다(영구 설정 + 1회성 환경변수 오버라이드).
4. 카탈로그가 없거나 키가 빠져도 도구가 깨지지 않는다(영문 폴백).

## 제약 — 코드에서 확인한 사실

| 사실 | 근거 | 설계 함의 |
|---|---|---|
| stock macOS는 **Bash 3.2.57** | `/bin/bash --version` | **`declare -A`(연관배열) 사용 금지.** 평범한 스칼라 변수 카탈로그로 구현 |
| `cc-tg.sh` 셰뱅 `#!/usr/bin/env bash`, 버전 가드 없음 | `cc-tg.sh:1` | bash 4 기능 가정 불가 |
| 출력 문자열이 한국어로 하드코딩 | `cc-tg.sh` 전반 `echo`/`printf` | 문자열을 키로 추출하는 전면 리팩터 필요 |
| 설치 매니페스트 `~/.config/cctg/install.conf`는 `> "$MANIFEST"` 로 **매번 전체 덮어쓰기**(기존 값 미보존), `update`가 install.sh 재실행 | `install.sh:169~180`, `cc-tg.sh` update 분기 | 언어 설정은 매니페스트와 **분리된 사용자 설정 파일**에 둔다(아래 "설정 저장 위치") |
| 자동완성을 `$DATA_DIR`(`~/.local/share`)로 **복사** 설치하는 선례 존재 | `install.sh:35,122~136` | 카탈로그 파일도 동일 패턴으로 `$DATA_DIR/cctg/`에 배치하면 "복사 설치 후 레포 삭제해도 동작" 원칙 유지 |

## 언어 결정 우선순위

런타임 시작 시 아래 순서로 언어를 결정한다(위가 우선).

1. **`CCTG_LANG` 환경변수** — 1회성 오버라이드. 예: `CCTG_LANG=en cctg status`
2. **사용자 설정 파일** `~/.config/cctg/config` 의 `lang=` 값
3. **로케일 자동 감지** — `$LC_ALL` → `$LANG` 순으로 보고 `ko*`면 `ko`, 그 외 `en`
4. **기본값 `en`**

지원 값은 `en`/`ko` 둘로 시작하고, 카탈로그 파일을 추가하면 자연히 확장된다.

## 메시지 카탈로그 구조

Bash 3.2 호환을 위해 **연관배열 대신 스칼라 변수 카탈로그 파일**을 쓴다.

```
messages/
  en.sh      # CCTG_MSG_* 영문 정의
  ko.sh      # CCTG_MSG_* 한글 정의
```

- 각 메시지는 **printf 템플릿**을 담은 변수로 정의한다(자리표시자 보존):

  ```sh
  # messages/ko.sh
  CCTG_MSG_REGISTERED='등록 완료: %s → cwd=%s, state=%s'
  CCTG_MSG_NOT_REGISTERED='ERROR: 등록되지 않은 프로젝트: %s'
  CCTG_MSG_STATUS_GLOBAL='전역 봇: %s (이 스크립트는 관리하지 않음)'
  ```
  ```sh
  # messages/en.sh
  CCTG_MSG_REGISTERED='Registered: %s → cwd=%s, state=%s'
  CCTG_MSG_NOT_REGISTERED='ERROR: not a registered project: %s'
  CCTG_MSG_STATUS_GLOBAL='Global bot: %s (not managed by this script)'
  ```

- `cc-tg.sh` 는 시작 시 **영문 카탈로그를 먼저 source(폴백 기준)**, 이어 선택 언어 카탈로그를 source 해 덮어쓴다. 이러면 특정 언어에서 키가 빠져도 영문으로 자동 폴백된다.
- 호출부는 `printf "$CCTG_MSG_REGISTERED\n" "$NAME" "$CWD" "$SD"` 형태. 헬퍼를 둘 거면 얇게:

  ```sh
  # 키 이름으로 템플릿을 찾아 출력(없으면 키 자체를 표기 → 누락 즉시 가시화)
  t() { local k="$1"; shift; local v="CCTG_MSG_$k"; printf "${!v:-$k}\n" "$@"; }
  ```

  `${!v}`(간접 확장)는 Bash 3.2에서 동작한다.

**카탈로그 배치/로딩**

| 설치 모드 | 카탈로그 위치 | 로딩 |
|---|---|---|
| copy(기본) | install.sh가 `messages/*.sh`를 `$DATA_DIR/cctg/messages/`로 복사 | cc-tg.sh가 `$DATA_DIR/cctg/messages/<lang>.sh` source |
| `--dev`(symlink) | 레포의 `messages/*.sh` 그대로 | cc-tg.sh가 자기 위치(심볼릭 역추적)로 레포 경로를 찾아 source |
| 폴백 | 파일 없음 | cc-tg.sh 내부에 **최소 영문 기본 문자열** 임베드 → 절대 깨지지 않음 |

> 단일 파일 배포를 유지하고 싶으면 카탈로그를 cc-tg.sh 안에 임베드하는 대안도 가능하나, 파일 비대화 + 향후 언어 추가 부담이 크다. 자동완성 복사 선례가 있으므로 **외부 파일 + 영문 임베드 폴백**을 권장한다.

## 설정 저장 위치

- **사용자 설정 파일**: `~/.config/cctg/config` (XDG `$XDG_CONFIG_HOME/cctg/config`)
  ```
  lang=ko
  ```
- 설치 매니페스트(`install.conf`)와 **분리**한다. 매니페스트는 환경에서 재계산 가능한 사실(repo·mode·version·bindir 등)만 담고 매 설치 시 전체 덮어써지므로, 사용자 선호인 언어 설정을 섞지 않는다.
- install.sh는 이 파일을 **없을 때만 시드**(idempotent, 기존 값 미덮어쓰기). `cctg lang` 이 갱신한다. `update`는 이 파일을 건드리지 않는다.
- 파일이 없을 수 있으므로(구버전 설치 등) 런타임은 항상 자동감지/en 폴백을 갖춘다.

## 설치 시 선택 (install.sh)

- 플래그 추가: `--lang en|ko`
  - 지정 시: 그 값으로 `~/.config/cctg/config` 의 `lang=` 시드(또는 갱신).
  - 미지정 시: 파일이 없으면 로케일 자동 감지값으로 시드, 있으면 손대지 않음.
- 사용 예:
  ```bash
  ./install.sh --lang en          # 영문으로 설치
  ./install.sh --lang ko          # 한글로 설치
  ./install.sh                    # 자동 감지(ko 로케일→ko, 그 외→en)
  ```
- 인자 파싱은 기존 `for arg in "$@"; case` 루프(`install.sh:42~53`)에 분기 추가.

## 설치 후 전환 (cctg lang)

새 서브커맨드로 영구 전환을 제공한다.

```bash
cctg lang            # 현재 언어 + 출처(env/config/auto/default) 표시 (= lang show)
cctg lang en         # 영문으로 영구 전환 (~/.config/cctg/config 기록)
cctg lang ko         # 한글로 영구 전환
cctg lang clear      # 사용자 설정 제거 → 자동 감지/기본값으로 회귀
```

- 1회성 전환은 설정 변경 없이 환경변수로: `CCTG_LANG=en cctg status`
- `cctg help`/`usage`·자동완성 후보·`doctor`에 `lang` 노출.

## cc-tg.sh 리팩터 범위

- 모든 사용자 노출 `echo`/`printf` 문자열을 키로 추출(대략 60~80개: `usage`, 등록/해제/이름변경, up/down/restart, status, doctor, update, 에러 메시지 등).
- 시작부에 **언어 결정 + 카탈로그 source** 초기화 블록 추가.
- `add` 의 대화형 프롬프트 문자열도 카탈로그화.
- 비노출(주석·내부 변수)은 그대로 둔다.

## 테스트·유지보수 고려

- **키 패리티 린트**: `en.sh`/`ko.sh` 가 같은 `CCTG_MSG_*` 키 집합을 갖는지 검사하는 작은 체크를 CI(또는 `doctor`)에 추가 — 번역 누락 드리프트 방지. (TODO의 shellcheck CI와 함께 묶기 좋음)
- **기존 테스트 영향**: 출력 문자열을 grep 으로 단언하는 샌드박스 테스트가 있으면, 로케일에 따라 깨진다. 테스트는 **`CCTG_LANG=en`(또는 ko) 고정** 후 안정적 토큰으로 단언하도록 수정한다.
- `${!v}` 간접 확장·`source` 경로는 Bash 3.2에서 검증한다(`/bin/bash`로 직접 실행).

## 문서 영향

- README의 `console` 예시 출력이 언어에 종속된다. 영문 `README.md`는 `en` 출력, `README.ko.md`는 `ko` 출력으로 각각 정합화한다(현재는 양쪽 다 한국어 실측 출력 + 영문 README에 안내 노트).
- 본 i18n 도입 후 그 안내 노트는 제거하고 실제 영문 출력으로 교체한다.

## 단계별 작업 순서

1. `messages/en.sh`·`messages/ko.sh` 신설 + 키 정의(현 한국어 출력 → 키 매핑표 먼저 작성).
2. `cc-tg.sh`: 언어 결정 블록 + `t()` 헬퍼 + 카탈로그 source + 영문 임베드 폴백.
3. `cc-tg.sh` 출력부를 키 호출로 일괄 치환.
4. `cctg lang` 서브커맨드 + `usage`/자동완성 갱신.
5. `install.sh`: `messages/*.sh` 복사(+symlink 모드 경로) + `--lang` 플래그 + config 시드.
6. 키 패리티 체크 + 기존 테스트 `CCTG_LANG` 고정.
7. README 예시 정합화 + 안내 노트 제거 + CHANGELOG 기록.
