## 배경

현재 `up`/`down`/`restart` 는 인자 1개(`<name>` 또는 `all`)만 처리한다. 여러 봇을 한 번에 다루려면 명령을 반복 입력해야 한다. 여러 타겟을 한 번에 받아 순차 자동 처리하도록 확장한다.

## 기능 요구 (FR)

- **FR-001**: `up`/`down`/`restart` 가 1개 이상의 타겟을 받아 **좌→우 순서로 순차** 처리한다. 타겟은 프로젝트 봇 이름·예약어 채널명(`telegram`/`discord`)·`all` 을 혼합할 수 있고, 각 타겟은 인자별로 라우팅된다(예약어→전용 경로, `all`→레지스트리 전체 전개).
- **FR-002 (continue-on-error)**: 한 타겟이 실패해도 중단하지 않고 다음 타겟을 계속 처리한다.
- **FR-003 (요약 출력)**: 2개 이상 타겟을 처리한 경우, 각 타겟의 성공/실패 정보를 종합한 요약 1줄을 마지막에 출력한다(성공 수·실패 수·실패 타겟명). 단일 타겟은 기존 per-target 출력으로 충분하므로 요약을 생략한다.
- **FR-004 (종료코드)**: 하나라도 실패하면 비0, 전부 성공이면 0 으로 종료한다.
- **FR-005 (하위호환, P-005)**: 단일 타겟·`all` 의 기존 동작·출력은 변하지 않는다.
- **FR-006 (자동완성/문서)**: bash/zsh 자동완성이 up/down/restart 의 **모든 타겟 위치**에서 이름·`all` 을 보완한다. usage 메시지·docs 의 사용법을 `<name...|all>` 로 갱신한다.
- **FR-007 (인자 0개)**: 타겟이 0개면 명확한 에러 + usage 를 출력하고 비0 종료한다.

## 비기능 요구 (NFR)

- **NFR-001 (P-001)**: Bash 3.2 호환(연관배열 미사용, `< <(...)` 프로세스치환은 기존 코드와 동일 패턴이라 허용).
- **NFR-002 (i18n)**: 신규 메시지는 en/ko 동일 키로 추가(`check-i18n-keys.sh` 패리티 통과).
- **NFR-003 (P-005 최소 표면)**: 새 서브커맨드·플래그 없이 기존 명령의 인자 수만 확장.

## 성공 기준 (SC)

- **SC-001**: `cctg up a b` 가 a·b 둘 다 기동(세션 2개 생성). → 테스트
- **SC-002**: `cctg down a b` 가 a·b 둘 다 정지. → 테스트
- **SC-003**: 한 타겟이 실패(미등록 등)해도 나머지 타겟은 처리되고, 종료코드는 비0. → 테스트
- **SC-004**: 2개 이상 처리 시 요약 라인(성공/실패 수)이 출력된다. → 테스트
- **SC-005**: 단일 타겟 동작·출력 불변(기존 테스트 회귀 0). → 기존 테스트
- **SC-006**: 예약어+프로젝트 혼합(`up <bot> telegram`)이 각각 라우팅된다. → 테스트(가능 범위)

## 범위 밖

- `logs`/`attach`: 단일 타겟 유지(attach 는 단일 세션 interactive, logs 는 단일 조회). 
- `status`(이미 전체 표시)·`config`(봇별 액션): 변경 없음.
- 병렬 실행(동시 기동): 범위 밖 — **순차** 처리만.
